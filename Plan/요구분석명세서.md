# Hair Salon 웹사이트 요구분석명세서 (SRS)

> 작성일: 2026-05-24
> 작업 범위: **웹 디자인 (프론트엔드 UI)** — 백엔드/외부 연동은 별도 단계에서 진행

---

## 1. 프로젝트 개요

### 1.1 목적
헤어 디자이너의 전문성을 신뢰감 있게 전달하고, 후기를 통해 시술 결과를 확인하며, 카카오톡 기반의 간편한 예약을 제공하는 헤어살롱 공식 웹사이트.

### 1.2 핵심 가치
- **신뢰**: 디자이너의 경력과 전문 분야를 명확히 보여준다.
- **검증**: 디자이너별 후기로 시술 결과를 미리 확인한다.
- **간편함**: 카카오톡 한 번으로 로그인·알림·예약이 끝난다.

### 1.3 사용자
| 구분 | 설명 | 주요 행동 |
|---|---|---|
| 일반 방문자 | 헤어살롱을 찾고 있는 잠재 고객 | 디자이너 탐색, 후기 열람, 예약 시도 |
| 예약 고객 | 카카오톡으로 로그인한 고객 | 예약 진행, 예약 확인, 알림 수신 |
| 어드민 | 헤어살롱 운영자 | 시술 메뉴 편집, 예약 현황 확인, 고객 정보 확인 |

## 2. 기능 요구사항

### 2.1 메인 페이지 (Home)
- **FR-H-01** 살롱의 분위기/컨셉을 보여주는 히어로 영역
- **FR-H-02** 대표 디자이너 프리뷰 (이름, 직급, 대표 작품 썸네일)
- **FR-H-03** 최근 후기 프리뷰 카드 (이미지 중심)
- **FR-H-04** 예약하기 CTA (상시 노출)

### 2.2 헤어 디자이너 정보 페이지
- **FR-D-01** 디자이너 목록 페이지 (그리드 카드)
- **FR-D-02** 디자이너 상세 페이지 — 다음 정보를 포함:
  - 프로필 사진
  - 이름 / 직급
  - **경력 연차**
  - **이전 근무지** (이전 살롱·브랜드)
  - **교육 이력** (강사 이력, 교육원, 자격)
  - **전문 시술 분야** (예: 다운펌, 발레아쥬, 베이비펌 등)
  - 대표 작품 갤러리
- **FR-D-03** 상세 페이지에서 "이 디자이너에게 예약하기" CTA
- **FR-D-04** 상세 페이지에서 "이 디자이너 후기 보기" 링크

### 2.3 후기 페이지
- **FR-R-01** 후기 전체 목록 페이지
- **FR-R-02** **카테고리 = 헤어 디자이너별** (세부 카테고리 없음)
  - 탭/필터: `전체` `디자이너 A` `디자이너 B` `디자이너 C` …
- **FR-R-03** 후기 카드: 시술 결과 이미지(전·후), 시술 종류, 별점/평점, 짧은 코멘트, 작성일, 담당 디자이너 태그
- **FR-R-04** 후기 상세 모달/페이지 — 전체 이미지 + 본문
- **FR-R-05** 후기 카드 클릭 시 해당 디자이너 페이지로 이동 가능

### 2.4 예약하기 페이지
- **FR-B-01** **카카오톡 로그인만 지원** (다른 로그인 수단 없음)
- **FR-B-02** 로그인 시 카카오톡 ID(또는 카카오 계정 식별자)와 핸드폰 번호를 받아 어드민에 저장
- **FR-B-03** 예약 폼 항목:
  1. **헤어 디자이너 선택** (디자이너별로 가능한 시술이 다름)
  2. **헤어 디자인 선택** — 선택한 디자이너가 가능한 시술 메뉴만 표시
     - 각 시술은 `이름`, `설명`, `가격`, `예상 소요시간` 포함
  3. **날짜·시간 선택** (캘린더 + 시간 슬롯)
  4. **요청사항** (선택 입력)
- **FR-B-04** 예약 완료 시 카카오톡으로 **예약 확정 메시지 즉시 전송**
- **FR-B-05** **예약일 1일 전 카카오톡 리마인더 메시지 자동 전송**
- **FR-B-06** 어드민은 예약자의 카카오톡 주소·핸드폰 번호 열람 가능

### 2.5 어드민 (관리자) 기능
> **참고**: 본 단계에서는 디자인 화면만 와이어프레임/Mock 수준으로 반영
- **FR-A-01** 시술(헤어 디자인) 메뉴 CRUD
  - 항목: 이름, 설명, 가격, 예상 소요시간, 이미지
- **FR-A-02** **디자이너별 시술 매핑** — 어떤 시술을 어떤 디자이너가 가능한지 설정
- **FR-A-03** 디자이너 프로필 CRUD (경력, 근무이력, 교육이력, 전문분야)
- **FR-A-04** 예약 목록 조회 (카카오 ID, 핸드폰 번호, 시술, 일시, 담당 디자이너)
- **FR-A-05** 기본 시술 메뉴는 시드 데이터로 제공 (커트, 펌, 염색, 클리닉 등)

## 3. 비기능 요구사항

| 코드 | 항목 | 요구사항 |
|---|---|---|
| NFR-01 | 반응형 | 모바일(360px) / 태블릿(768px) / 데스크탑(1280px+) |
| NFR-02 | 분위기 | 따뜻하고 프리미엄한 살롱의 무드. 사진 중심, 여백 충분 |
| NFR-03 | 접근성 | 명도 대비 WCAG AA, 키보드 포커스 표시 |
| NFR-04 | 성능 | 이미지 lazy-load, 첫 화면 LCP 2.5s 이하 목표 |
| NFR-05 | 언어 | 한국어 우선 |
| NFR-06 | 알림 | 카카오톡 메시지(알림톡) 1일 전 리마인더 발송 |

## 4. 데이터 모델 (디자인 단계 기준)

```
Designer {
  id, name, title, yearsOfExperience,
  profileImage, bio,
  workHistory[],        // 이전 근무지
  educationHistory[],   // 교육·강사 이력
  specialties[],        // 전문 시술 분야 (string)
  portfolioImages[]
}

Service {                // 헤어 디자인 시술 메뉴
  id, name, description, price, durationMin, image,
  availableDesignerIds[]  // 가능한 디자이너 매핑
}

Review {
  id, designerId, customerName(익명화), rating,
  beforeImage, afterImage,
  serviceType, comment, createdAt
}

Booking {
  id, kakaoId, phoneNumber,
  designerId, serviceId,
  reservedAt, requestNote,
  status                  // confirmed | reminded | done | canceled
}
```

## 5. 디자인 단계 산출물 (이번 단계 결과물)
- 디자인 토큰을 적용한 정적 HTML/CSS 페이지
  - 홈, 디자이너 목록, 디자이너 상세, 후기 목록, 예약하기, (어드민 화면 mock 1~2개)
- 카카오톡 로그인·메시지 발송 등 외부 연동은 **버튼/이벤트 자리만 마련** (실제 연동 X)

## 6. 디자인 컨셉 키워드
> 이후 선정할 `DESIGN.md`의 평가 기준
- **따뜻한 무드 (warm canvas)** — 차가운 화이트보다 크림·아이보리 톤
- **사진 중심** — 시술 결과·디자이너 인물 사진을 강조하는 레이아웃
- **부드러운 곡선** — 둥근 카드/필 모양 CTA (날카로운 모서리 지양)
- **에디토리얼 / 살롱 잡지** 느낌 — 여백, 단정한 서체
- **신뢰감** — 절제된 단일 액센트 컬러, 화려한 그라데이션 지양
