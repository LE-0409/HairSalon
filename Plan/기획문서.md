## 1. 사이트맵

```
HOME (/)
├── 디자이너 (/designers)
│   └── 디자이너 상세 (/designers/:id)
├── 후기 (/reviews)
│   └── 디자이너별 필터 (?designer=:id)
├── 예약하기 (/booking)
│   ├── 1) 카카오 로그인
│   ├── 2) 디자이너 선택
│   ├── 3) 시술(헤어 디자인) 선택
│   ├── 4) 날짜·시간 선택
│   └── 5) 확인 & 예약 완료
└── 어드민 (/admin)        ← Mock 화면
    ├── 예약 목록
    ├── 시술 메뉴 관리
    ├── 디자이너 관리
    └── 디자이너-시술 매핑
```

## 2. 페이지별 구성

### 2.1 HOME `/`
| 섹션 | 내용 |
|---|---|
| Global Nav | 로고 / 디자이너 / 후기 / 예약하기(주요 CTA) |
| Hero | 살롱 분위기 사진 + 카피 + "예약하기" CTA |
| Featured Designers | 디자이너 4~6명 카드 슬라이드 |
| Recent Reviews | 최근 후기 6~8장 (Before/After 이미지 강조) |
| About | 살롱 소개 한 문단 + 위치/영업시간 |
| Footer | SNS, 연락처, 주소 |

### 2.2 디자이너 목록 `/designers`
- 그리드 (3 col @ desktop, 2 col @ tablet, 1 col @ mobile)
- 카드: 인물 사진 / 이름 / 직급 / 경력 N년 / 전문 분야 태그 2~3개

### 2.3 디자이너 상세 `/designers/:id`
| 영역 | 내용 |
|---|---|
| Hero | 큰 프로필 사진 + 이름 + 직급 |
| 소개 (Bio) | 디자이너 자기소개 |
| 경력 | 연차 / 이전 근무지 리스트 |
| 교육 | 교육·강사 이력 리스트 |
| 전문 분야 | 태그 형태 (예: 다운펌, 발레아쥬…) |
| 포트폴리오 갤러리 | 작품 이미지 그리드 |
| CTA | "이 디자이너에게 예약" / "이 디자이너 후기 보기" |

### 2.4 후기 페이지 `/reviews`
| 영역 | 내용 |
|---|---|
| 카테고리 탭 | `전체` / 디자이너별 탭 (가로 스크롤) |
| 후기 카드 그리드 | Before/After 이미지, 시술명, 별점, 코멘트, 디자이너 이름 |
| 후기 상세 모달 | 클릭 시 큰 이미지 + 전체 본문 |

### 2.5 예약하기 `/booking` (5단계 스텝)
> 단계별 스텝 인디케이터 노출

| 단계 | 화면 |
|---|---|
| 1 | 카카오톡 로그인 안내 + 노란색 카카오 버튼 |
| 2 | 디자이너 선택 (그리드 카드, 단일 선택) |
| 3 | 시술 선택 (선택된 디자이너가 가능한 메뉴만 노출, 가격·시간 표시) |
| 4 | 날짜·시간 선택 (월별 캘린더 + 시간 슬롯) |
| 5 | 정보 확인 (디자이너·시술·일시·연락처) → "예약 확정" |
| 완료 | "예약이 완료되었습니다 / 카카오톡으로 확인 메시지를 보냈어요" |

### 2.6 어드민 Mock `/admin`
- 좌측 사이드바 + 우측 컨텐츠 영역
- 예약 목록 테이블 (예약일, 시술, 디자이너, 카카오 ID, 핸드폰)
- 시술 메뉴 카드 그리드 + 편집 모달
- 디자이너-시술 매핑 매트릭스 (체크박스)

## 3. 사용자 흐름 (User Flow)

### 3.1 예약 흐름
```
HOME → "예약하기" 클릭
    → [Step 1] 카카오 로그인
        → 카카오톡 동의 (ID, 휴대폰)
    → [Step 2] 디자이너 선택
    → [Step 3] 시술 선택 (디자이너에 따라 메뉴 다름)
    → [Step 4] 날짜·시간 선택
    → [Step 5] 확인 → "예약 확정"
    → 완료 화면 + 카카오톡 확정 메시지 발송
    → (예약 1일 전) 카카오톡 리마인더 메시지 자동 발송
```

### 3.2 디자이너 탐색 흐름
```
HOME → "디자이너" 메뉴 클릭 → 디자이너 목록
    → 카드 클릭 → 디자이너 상세
    → 후기 보기 → 해당 디자이너 후기 필터링 페이지
    → "이 디자이너에게 예약" → 예약 Step 3 (디자이너 미리 선택된 상태)
```

### 3.3 후기 탐색 흐름
```
HOME → "후기" 메뉴 → 전체 후기
    → 디자이너 탭 클릭 → 디자이너별 후기 필터
    → 후기 카드 클릭 → 상세 모달
```

## 4. 디자인/UX 원칙

1. **사진이 주인공이다** — 카피보다 시술 결과 이미지가 먼저 보여야 한다.
2. **여백을 두려워하지 않는다** — 잡지 같은 호흡.
3. **CTA는 단 하나의 색** — 예약 버튼만 강한 액센트. 나머지는 절제.
4. **부드러운 곡선** — 카드 14~20px radius, 버튼은 full pill.
5. **모바일 우선** — 예약은 모바일에서 끝나는 경우가 압도적이라 가정.
